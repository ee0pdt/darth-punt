# UI Tidy-Up Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the Tron-leaning UI redesign described in
`docs/superpowers/specs/2026-05-20-ui-tidy-up-design.md` — performance-first stage + drawer layout,
two-tier button family, real toggle switches and sliders, drop the purple palette and scanline.

**Architecture:** Pure UI refactor. The audio scheduler and state model are untouched. Changes are
concentrated in `index.html` (markup + the embedded `<style>` block), small updates to
`src/ui/controls.ts`, `src/ui/sequencer.ts`, `src/ui/readouts.ts`, `src/ui/events.ts`, plus a new
`src/ui/drawer.ts` mounted from `src/main.ts`. Each task lands a coherent slice that leaves the app
working.

**Tech Stack:** Vanilla TypeScript, Deno + esbuild build, no UI framework, no test harness. Per-task
verification = `deno task check` + `deno task build` + (optional) Chrome DevTools smoke load. Visual
sign-off comes from the user at the end.

**General per-task verification snippet** (every task uses this before commit):

```bash
deno task check && deno task build
```

Expected: `check` exits 0 (fmt, lint, type-check clean), `build` writes `dist/main.js` without
errors.

## File Structure (locked in here)

- `index.html` — markup + `<style>` block. Receives the bulk of changes across multiple tasks.
- `src/ui/readouts.ts` — small text updates for the new inline header readout format.
- `src/ui/sequencer.ts` — mute button class updates only; grid render logic unchanged.
- `src/ui/controls.ts` — slider handlers update a new `--pct` CSS variable; pattern dropdown wiring
  replaces 4-button wiring; toggle-switch event delegation replaces `.tag-btn` button toggling (same
  state mutation, new markup).
- `src/ui/events.ts` — no logic change; tension overlay class toggle stays as-is (the colour change
  is CSS-only).
- `src/ui/drawer.ts` — **new file**. Owns drawer open/close, tab switching, mobile bottom-sheet
  swipe-to-close.
- `src/main.ts` — single new line calling `mountDrawer()`.

No changes to `src/audio/*`, `src/state.ts`, `src/patterns.ts`, `src/visual/*`.

```typescript
/**
 * Drawer mounting: open/close + tab switching for the controls drawer.
 * Behaviour is identical on desktop and mobile at this stage; the
 * bottom-sheet sliding behaviour for narrow viewports is added later.
 */

type TabId = "mix" | "auto" | "patterns";

export function mountDrawer(): void {
  const handle = document.getElementById("drawerHandle");
  const drawer = document.getElementById("drawer");
  const closeBtn = document.getElementById("drawerClose");
  if (!handle || !drawer) return;

  handle.addEventListener("click", () => toggleDrawer(drawer, handle));
  closeBtn?.addEventListener("click", () => setDrawerOpen(drawer, handle, false));

  document.querySelectorAll<HTMLElement>(".drawer-tab").forEach((tab) => {
    tab.addEventListener("click", () => {
      const id = tab.dataset.tab as TabId | undefined;
      if (id) selectTab(id);
    });
  });
}

function toggleDrawer(drawer: HTMLElement, handle: HTMLElement): void {
  setDrawerOpen(drawer, handle, !drawer.classList.contains("open"));
}

function setDrawerOpen(drawer: HTMLElement, handle: HTMLElement, open: boolean): void {
  drawer.classList.toggle("open", open);
  drawer.setAttribute("aria-hidden", open ? "false" : "true");
  handle.setAttribute("aria-expanded", open ? "true" : "false");
}

function selectTab(id: TabId): void {
  document.querySelectorAll<HTMLElement>(".drawer-tab").forEach((t) => {
    const match = t.dataset.tab === id;
    t.classList.toggle("active", match);
    t.setAttribute("aria-selected", match ? "true" : "false");
  });
  document.querySelectorAll<HTMLElement>(".drawer-pane").forEach((p) => {
    p.classList.toggle("active", p.dataset.pane === id);
  });
}
```

- [ ] **Step 10.2: Add swipe-to-close to mobile sheet in drawer.ts**

Open `src/ui/drawer.ts`. Below the existing `mountDrawer` function, add a small touch handler that
closes the drawer when the user swipes down from the tab strip area. Update `mountDrawer` to wire

Add at the bottom of the file:

```typescript
function attachSheetSwipe(drawer: HTMLElement, handle: HTMLElement): void {
  const tabs = drawer.querySelector<HTMLElement>(".drawer-tabs");
  if (!tabs) return;

  let startY: number | null = null;

  tabs.addEventListener("touchstart", (ev) => {
    if (!drawer.classList.contains("open")) return;
    if (!matchMedia("(max-width: 720px)").matches) return;
    const t = ev.touches[0];
    if (t) startY = t.clientY;
  }, { passive: true });

  tabs.addEventListener("touchmove", (ev) => {
    if (startY === null) return;
    const t = ev.touches[0];
    if (!t) return;
    const dy = t.clientY - startY;
    if (dy > 60) {
      drawer.classList.remove("open");
      drawer.setAttribute("aria-hidden", "true");
      handle.setAttribute("aria-expanded", "false");
      startY = null;
    }
  }, { passive: true });

  tabs.addEventListener("touchend", () => {
    startY = null;
  }, { passive: true });
}
```

Update `mountDrawer` so it calls the new helper:

```typescript
export function mountDrawer(): void {
  const handle = document.getElementById("drawerHandle");
  const drawer = document.getElementById("drawer");
  const closeBtn = document.getElementById("drawerClose");
  if (!handle || !drawer) return;

  handle.addEventListener("click", () => toggleDrawer(drawer, handle));
  closeBtn?.addEventListener("click", () => setDrawerOpen(drawer, handle, false));

  document.querySelectorAll<HTMLElement>(".drawer-tab").forEach((tab) => {
    tab.addEventListener("click", () => {
      const id = tab.dataset.tab as TabId | undefined;
      if (id) selectTab(id);
    });
  });

  attachSheetSwipe(drawer, handle);
}
```
